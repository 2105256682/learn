## 一、爬虫的基础知识

### 1. 什么是爬虫？

**爬虫**（Web Crawler / Spider），又称为网络爬虫、网页蜘蛛，是一种按照一定规则，自动地抓取互联网信息的程序或脚本。

简单理解：爬虫就是模拟人在浏览器中访问网页的行为，自动把网页上的数据“爬”下来保存到本地。

#### 核心特征

| 特征 | 说明 |
| :--- | :--- |
| **自动化** | 无需人工逐页打开，程序自动发起请求并处理响应 |
| **批量性** | 可以同时抓取大量网页或大量数据 |
| **目标性** | 根据 URL 规则、页面结构、关键词等提取目标信息 |
| **可存储** | 将数据保存到文件、数据库或进一步分析处理 |

#### 生活中的类比

| 类比 | 爬虫对应 |
| :--- | :--- |
| 图书馆管理员整理书籍 | 爬虫按目录抓取并分类网页 |
| 搜索引擎收录网页 | 百度/Google 爬虫 24 小时不间断爬取互联网 |
| 价格监控助手 | 爬虫定时抓取商品价格并提醒降价 |

### 2. 为什么要学习爬虫？

学习爬虫，本质上是在学习**“如何从互联网上高效获取并利用数据”**。在大数据与人工智能时代，数据是最核心的资源之一，而爬虫是获取公开网络数据最直接、最基础的技术手段。

#### 学习爬虫的主要价值

| 价值 | 具体说明 |
| :--- | :--- |
| **数据获取能力强** | 可以快速获取新闻、商品、评论、股票、论文、招聘等海量公开数据 |
| **提升工作效率** | 把重复的人工复制粘贴工作交给程序自动完成 |
| **为数据分析打基础** | 爬虫是数据采集环节，后续可结合 Pandas、可视化、机器学习进行分析 |
| **就业机会广泛** | 爬虫工程师、数据工程师、Python 后端开发等岗位常见需求 |
| **理解网络原理** | 学习 HTTP、HTML、正则、XPath、JSON 等 Web 核心技术 |

#### 常见学习场景

```text
- 想批量下载壁纸/小说/视频？→ 爬虫可以自动下载
- 想监控某商品价格变化？→ 爬虫定时抓取并对比
- 想做论文文献分析？→ 爬虫批量抓取标题、摘要、引用
- 想做舆情分析？→ 爬虫抓取社交媒体评论和新闻
```

### 3. 爬虫的用途？

爬虫的用途非常广泛，几乎覆盖了所有需要从互联网获取数据的领域。

| 应用领域 | 典型用途 | 示例数据 |
| :--- | :--- | :--- |
| **搜索引擎** | 收录网页，建立索引 | 网页标题、正文、链接关系 |
| **电商分析** | 商品价格监控、竞品分析 | 商品名称、价格、销量、评论 |
| **舆情监控** | 新闻、社交媒体情绪分析 | 微博评论、新闻标题、转发数 |
| **招聘求职** | 岗位信息聚合、薪资分析 | 岗位名称、薪资、公司、要求 |
| **金融投资** | 股票行情、基金数据、宏观数据 | 股价、成交量、财务报表 |

> ⚠️ **注意**：爬虫只能抓取**公开合法**的数据，不得侵犯隐私、著作权，也不得对目标网站造成过大访问压力。

### 4. 爬虫的分类？

根据爬虫的目标、范围、策略等不同，可以将爬虫分为多种类型。

#### 4.1 按目标范围分类

| 类型 | 定义 | 特点 | 典型代表 |
| :--- | :--- | :--- | :--- |
| **通用爬虫** | 尽可能抓取互联网上所有网页 | 抓取范围广、数据量大、不针对特定内容 | 百度蜘蛛、Googlebot |
| **聚焦爬虫** | 只抓取特定主题或特定网站的数据 | 目标明确、精准、去重要求高 | 某电商商品爬虫、某新闻站爬虫 |
| **增量爬虫** | 只抓取新增或发生变化的网页 | 节省资源、更新快 | 新闻资讯聚合爬虫 |
| **深层爬虫** | 抓取表单、登录后才能访问的页面 | 需要模拟登录、处理验证码 | 社交平台数据爬虫 |

#### 4.2 按实现方式分类

| 类型 | 说明 | 常用技术 |
| :--- | :--- | :--- |
| **静态页面爬虫** | 直接请求 HTML 即可获得完整数据 | `requests` + `BeautifulSoup` / `re` |
| **动态页面爬虫** | 数据由 JavaScript 异步加载，需要渲染页面 | `Selenium` / `Playwright` / `requests` 分析 API |
| **API 接口爬虫** | 直接抓取网站提供的 JSON 接口 | `requests` + JSON 解析 |
| **App 爬虫** | 抓取移动端 App 数据，需要抓包分析 | `Charles`、`Fiddler`、`mitmproxy` |

#### 4.3 对比记忆表

```
通用爬虫像撒网，聚焦爬虫像钓鱼；
增量爬虫只追新，深层爬虫钻到底。
```

| 对比项 | 通用爬虫 | 聚焦爬虫 |
| :--- | :--- | :--- |
| 目标 | 全互联网 | 特定主题/网站 |
| 数据量 | 海量 | 相对可控 |
| 精准度 | 低 | 高 |
| 去重策略 | 复杂 | 较简单 |
| 资源消耗 | 大 | 小 |

### 5. 爬虫的工作流程

一个标准爬虫的工作流程通常可以概括为 **4 个核心步骤**：

1. **发送请求**：爬虫首先发送 HTTP 请求到目标网站
2. **获取响应**：获取请求返回的响应内容
3. **解析响应，提取数据**：爬虫解析响应内容，提取需要的信息，比如 URL 链接、文本数据等
4. **存储数据**：爬虫将提取的信息存储到本地文件或数据库中

#### 5.1 流程图解

```text
┌─────────────┐
│   URL 队列   │
└──────┬──────┘
       │ url
       ▼
┌─────────────────────┐
│  1. 发送请求获取响应   │
└──────┬──────────────┘
       │ response
       ▼
┌─────────────────────┐
│  2. 解析响应          │
│  3. 提取数据          │
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│  4. 存储数据          │
│    (文件/数据库)      │
└──────┬──────────────┘
       │ 新 URL
       └──────────────► URL 队列（循环操作）
```

#### 5.2 各步骤详细说明

| 步骤 | 任务说明 | 涉及技术/工具 |
| :--- | :--- | :--- |
| **发送请求** | 使用 HTTP 协议向目标 URL 发起 GET/POST 请求 | `requests`、`urllib`、`aiohttp` |
| **获取响应** | 接收服务器返回的 HTML、JSON、二进制文件等 | 状态码、响应头、编码处理 |
| **解析响应** | 从响应内容中定位并提取目标数据 | `BeautifulSoup`、`lxml`、`re`、XPath |
| **提取数据** | 把解析出的数据整理成结构化形式 | 字典、列表、DataFrame |
| **存储数据** | 保存到本地文件或数据库，便于后续使用 | `.txt`、`.csv`、`.json`、MySQL、MongoDB |
| **循环处理** | 发现新 URL 后继续抓取，形成闭环 | URL 队列、去重、调度 |

#### 5.3 代码示例：一个极简爬虫

```python
import requests
from bs4 import BeautifulSoup

# 1. 发送请求
url = "https://example.com"
response = requests.get(url)

# 2. 获取响应内容
html = response.text
print(f"状态码: {response.status_code}")

# 3. 解析响应，提取数据
soup = BeautifulSoup(html, "html.parser")
title = soup.find("title").get_text()
links = [a.get("href") for a in soup.find_all("a")]

print(f"网页标题: {title}")
print(f"链接数量: {len(links)}")

# 4. 存储数据
with open("result.txt", "w", encoding="utf-8") as f:
    f.write(f"标题: {title}\n")
    f.write("链接:\n")
    for link in links:
        f.write(f"{link}\n")
```

#### 5.4 工作流程口诀

```
请求响应解析存，
URL 队列循环跟；
数据提取要精准，
合法合规记在心。
```

### 6. robots 协议

#### 6.1 什么是 robots 协议？

**Robots 协议**（也称为 `robots.txt`）是一个位于网站根目录下的文本文件，用于指示搜索引擎爬虫哪些页面可以访问，哪些页面不应该被访问。该文件包含一系列规则，定义了爬虫对网站的访问权限。

> 它本质上是一份“君子协定”，对爬虫行为进行道德与法律层面的约束，而非技术强制限制。

#### 6.2 基本语法

Robots 协议的基本语法包括两个关键字：**User-agent** 和 **Disallow**。

| 关键字 | 作用 | 示例 |
| :--- | :--- | :--- |
| **User-agent** | 指定爬虫的名称或标识符，表示规则适用于哪个爬虫 | `User-agent: *` 表示所有爬虫 |
| **Disallow** | 指定不允许被访问的 URL 路径 | `Disallow: /admin/` 表示禁止访问 admin 目录 |

#### 6.3 常见 robots.txt 示例

```txt
# 允许所有爬虫访问所有页面
User-agent: *
Disallow:
```

```txt
# 禁止所有爬虫访问任何页面
User-agent: *
Disallow: /
```

```txt
# 禁止所有爬虫访问 /admin/ 和 /private/ 目录
User-agent: *
Disallow: /admin/
Disallow: /private/

# 允许百度爬虫访问全部页面
User-agent: Baiduspider
Disallow:
```

#### 6.4 案例网站

| 网站 | robots.txt 地址 |
| :--- | :--- |
| 百度 | `http://www.baidu.com/robots.txt` |
| 新浪 | `http://www.sina.com/robots.txt` |
| 腾讯 | `http://www.qq.com/robots.txt` |
| 淘宝 | `http://www.taobao.com/robots.txt` |

#### 6.5 查看 robots.txt 的 Python 代码

```python
import requests

urls = [
    "https://www.baidu.com/robots.txt",
    "https://www.sina.com/robots.txt",
    "https://www.qq.com/robots.txt",
    "https://www.taobao.com/robots.txt",
]

for url in urls:
    try:
        resp = requests.get(url, timeout=10)
        print(f"\n===== {url} =====")
        print(resp.text[:500])  # 只打印前 500 字符
    except Exception as e:
        print(f"{url} 请求失败: {e}")
```

#### 6.6 遵守 robots 协议的重要性

| 原因 | 说明 |
| :--- | :--- |
| **法律合规** | 违反网站声明抓取受限数据可能涉及法律风险 |
| **保护服务器** | 避免对目标网站造成过大访问压力，甚至被封禁 IP |
| **职业操守** | 尊重网站所有者的意愿，做负责任的爬虫开发者 |
| **降低反爬** | 遵守规则可减少被反爬机制拦截的概率 |

> 💡 **提示**：在开始学习爬虫时，建议先在本地或测试网站上练习，熟练掌握后再抓取真实网站，并始终遵守 `robots.txt` 和相关法律法规。

## 二、核心概念对比表

| 概念 | 一句话解释 | 注意点 |
