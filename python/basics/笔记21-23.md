## 一、私有属性和函数（DAY21）

### 1. 访问权限控制

在 Python 中，通过下划线前缀控制属性和方法的访问可见性：

| 写法 | 可见性 | 能否继承 |
|---|---|---|
| `name` | 公开（public） | ✅ 可以继承 |
| `_name` | 受保护（protected） | ✅ 可以继承，约定不直接访问 |
| `__name` | 私有（private） | ❌ 不能继承，类外无法直接访问 |

### 2. 私有属性/方法定义

```python
class Animal:
    __name__ = "动物"       # 私有类属性

    def __init__(self, color):
        self.__color = color  # 私有实例属性

    def __run(self):        # 私有方法
        print("动物跑起来了")

    def say(self):
        print('动物喊叫')
        print(self.__color)  # 类内可以访问私有属性

    def set_color(self, new_color):  # setter
        self.__color = new_color

    def get_color(self):             # getter
        return self.__color

class Dog(Animal):
    pass


# 使用
d = Dog('red')
# d.__run()   # ❌ AttributeError！私有方法无法类外调用
d.set_color('blue')  # ✅ 通过公共方法修改私有属性
print(d.get_color()) # ✅ 通过公共方法访问私有属性
d.say()
```

> **注意**：双下划线开头的属性/方法**不能继承给子类**，且**不能在类外部直接调用和访问**。

### 3. 强制访问私有属性（不推荐）

```python
# 名称改写（name mangling）机制
print(d._Animal__color)  # 可以强制访问，但破坏封装原则
```

### 4. getter / setter 规范

| 函数名 | 用途 |
|---|---|
| `get_xx()` | 获取私有属性值 |
| `set_xx(val)` | 修改私有属性值 |

> Python 哲学："We are all consenting adults here"，开放比封闭好，私有化非必须。实际开发中很多 Python 程序员选择直接使用公开属性。

## 二、导入模块和 Python 包（DAY22）

### 1. 五种导入方式

```python
# 方式一：导入整个模块
import math
print(math.log2(8))       # 模块名.函数名

# 方式二：导入模块所有内容（不推荐，易冲突）
from math import *
print(log2(8))            # 直接使用函数名

# 方式三：精确导入（推荐）
from math import log2, log10
print(log2(8))
print(log10(100))

# 方式四：别名导入
import multiprocessing as mp
print(mp.current_process())

# 方式五：函数别名
from math import log2 as lg2
print(lg2(8))
```

### 2. 导入注意事项

1. **同名冲突**：使用 `from .. import ..` 或 `from .. import *` 导入多个模块时，若模块内有同名功能，**调用的是后面导入的模块的功能**。

2. **搜索顺序**：Python 解析器对模块位置的搜索顺序：
   - 当前目录
   - 不在当前目录时，搜索 `PYTHONPATH` 环境变量下的每个目录（`sys.path` 可查看）

3. **文件名冲突**：自己的文件名不要和已有模块名重复（如不要叫 `random.py`），否则会导致模块功能无法使用。

### 3. `__all__` 限制导入

```python
# my_module.py
__all__ = ['testA']  # 声明：from my_module import * 时只能导入 testA

def testA():
    print('testA')

def testB():
    print('testB')  # 不会被 * 导入
```

> `__all__` 只对 `from xxx import *` 生效，`import 模块名` 的方式不受影响。

### 4. 测试模块

```python
# 测试代码放在 if __name__ == '__main__': 下
# 当模块被直接运行时执行，被导入时不执行
if __name__ == '__main__':
    print('测试内容')
```

### 5. Python 的包（Package）

- **包**：由模块文件构成，将众多具有相关功能的模块结构化组合形成包。
- **新建包后**，包内部会自动创建 `__init__.py` 文件，这个文件控制包的导入行为。
- 在 `__init__.py` 中添加 `__all__ = [模块1, 模块2]`，控制允许导入的模块列表（只在 `from xxx import *` 时有效）。

**目录结构示例：**
```
pkg/
├── __init__.py
├── sub_pkg1/
│   ├── __init__.py
│   ├── mod1.py
│   └── mod2.py
```

### 6. 导入包中的模块

```python
import 包名.模块名
from 包名.模块名 import *
```

## 三、异常和 Debug 调试（DAY23）

### 1. 异常处理基本结构

```python
try:
    # 可能发生错误的代码
    f = open('abc.txt')
    s = f.readline()
    num = int(s.strip())
    print(num)
except 异常类型1:
    # 捕获到异常类型1执行的代码
    pass
except 异常类型2:
    # 捕获到异常类型2执行的代码
    pass
except Exception as e:
    # Exception 是所有异常的父类，可以捕获所有异常
    print(e)
```

> **重要**：如果尝试执行的代码的异常类型和要捕获的异常类型不一致，则无法捕获异常。

### 2. try 语句工作原理

1. **首先**，执行 `try` 子句（`try` 和 `except` 关键字之间的语句）。
2. **如果没有触发异常**，则跳过 `except` 子句，`try` 语句执行完毕。
3. **如果在执行 try 子句时发生异常**，则跳过该子句中剩下的部分。如果异常的类型与 `except` 关键字后指定的异常相匹配，则执行 `except` 子句，然后跳到 `try/except` 代码块之后继续执行。
4. **如果发生的异常与 except 子句中指定的异常不匹配**，则它会被传递到外部的 `try` 语句中；如果没有找到处理程序，则它是一个**未处理异常**且执行将终止。

### 3. 强制抛出异常（raise）

```python
try:
    raise NameError('HiThere')  # 主动抛出指定异常
except NameError:
    print('An exception flew by!')
```

> `raise` 语句支持强制触发指定的异常。如果只想判断是否触发了异常，但并不打算处理该异常，可以使用更简单的 `raise` 语句重新触发异常。

### 4. 异常中的 else

`else` 表示的是**如果没有异常**要执行的代码。

```python
try:
    print(1)
except Exception as result:
    print(result)
else:
    print('我是 else，是没有异常的时候执行的代码')
```

### 5. Finally 语句

`finally` 表示的是**无论是否异常都要执行**的代码，例如关闭文件。一般都是做一些清理工作。

```python
try:
    f = open('test.txt', 'r')
except Exception as result:
    f = open('test.txt', 'w')
else:
    print('没有异常，真开心')
finally:
    f.close()  # 无论是否异常，都会执行关闭文件
```

### 6. 自定义异常类

```python
class PasswordToShortError(Exception):
    """自定义的异常：密码太短了"""

    def __init__(self, length, min_len):
        self.length = length
        self.min_len = min_len

    # 打印异常对象，就会打印 __str__ 函数的返回值
    def __str__(self):
        return f'你输入的密码长度是{self.length}，密码不能少于{self.min_len}'


def input_password():
    pwd = input('请输入你的密码:')
    if len(pwd) < 6:
        raise PasswordToShortError(len(pwd), min_len=6)


if __name__ == '__main__':
    try:
        input_password()
    except Exception as err:
        print(err)
```

### 7. 完整异常处理流程图

```
        try:
            ├─ 正常执行 → else → finally
            └─ 发生异常 → except 匹配 → finally
                 └─ 不匹配 → 向上抛出
```

### 8. 异常处理速查表

| 关键字 | 作用 | 执行条件 |
|---|---|---|
| `try` | 放置可能出错的代码 | 必须 |
| `except` | 捕获并处理异常 | 异常发生时 |
| `else` | 无异常时执行的代码 | try 无异常 |
| `finally` | 无论是否异常都执行 | 总是执行 |
| `raise` | 主动抛出异常 | 手动触发 |

### 9. 常见异常类型

| 异常类型 | 触发场景 |
|---|---|
| `FileNotFoundError` | 文件不存在 |
| `ValueError` | 值类型错误（如 int('abc')） |
| `TypeError` | 类型不匹配 |
| `IndexError` | 索引超出范围 |
| `KeyError` | 字典键不存在 |
| `ZeroDivisionError` | 除以零 |
| `AttributeError` | 对象没有该属性 |
| `Exception` | 所有常规异常的父类 |
